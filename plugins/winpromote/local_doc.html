Promote Windows NT, 200x, XP, Vista, Seven and 10 user to Administrator - Rescapp - Documentation

Problems that are usually solved

  • Scenario 1: Your former sysadmin disappears

    Your former sysadmin is no longer there. It doesn't matter if you have fired him and he's angry at you or if he had a fatal car accident. You need to take in control of your Business Windows server. You are also lucky because you have an account in your Business Windows server which you can login to.

What do you get after using the option?

If everything goes ok the next time you boot your computer when Windows login appears you will be able to login with your account. Now thanks to your account you can run programs which only Windows administrators can usually run.

Steps

  • Step 1

    Rescapp searches for Windows system partitions, on your disk.
  • Step 2

    If such partitions have been found, Rescapp displays a list of the partitions. At the same time you are prompted:

    Which Windows partition?

    You are supposed to choose, your Windows partition, (you will probably only see one), and press the OK button.

  • Step 3

    Rescapp tries to backup your Windows configuration files, on your Windows sytem.
  • Step 4

    You are optionally informed about the Windows configuration files backup process.

    You will get (if an error happens):

    [ERROR] Backup of SAM directory went wrong.

  • Step 5

    Rescapp searches for System users, based on previous information.
  • Step 6

    You are prompted:

    Choose Windows user to promote to Admin

    You are supposed to select, the user which you want to, promote to Administrator.

  • Step 7

    You are informed about, the promote Windows NT, 200x, XP, Vista, Seven and 10 user to Administrator process.

    You either get:

    [SUCCESS] The user was promoted to Admin OK! :)

    or

    [ERROR] The user was not promoted to Admin. Something went wrong! :(
