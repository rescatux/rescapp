Restore generic MBR code so that Windows boots again - Rescapp - Documentation
  • Step 1

    Rescapp searches for primary partitions on your drive.
  • Step 2

    If such partitions have been found Rescapp displays a list of these partitions. At the same time you are prompted:

    Which partition?

    You are supposed to choose your Windows (you will probably only see one) and press the OK button. If you have several Windows partitions, the best Windows partition to choose, is the first one that you installed on your computer.

  • Step 3

    Rescapp searches for hard drives present on your system based on the information found in the /proc/partitions file.
  • Step 4

    You are prompted:

    Which hard drive do you want to install the Windows mbr on?

    You are supposed to take a look at hard drive sizes (to be able to identify your hard drives) and choose (usually) the one that it is set to be the first one to boot in your BIOS settings.

  • Step 5

    You are prompted:

    What is the correct position for this hard drive?

    Just select the right boot order position for the prompted hard drive according to your BIOS settings.

  • Step 6

    Rescapp with all gathered information, is going to set up a Windows mbr on your chosen hard disk so that the partition you have chosen is booted at machine boot.Please wait. Currently (Rescatux 0.23) this step is time consuming, but no Please wait message has been implemented.
  • Step 7

    You are informed about the mbr installation process.

    You either get:

    Success! Windows mbr was installed OK! :)

    or

    Failure! Windows mbr was not installed. Something went wrong! :(

  • Step 8

    A log file called winmbr.log.txt can be found in the Desktop logs folder.
