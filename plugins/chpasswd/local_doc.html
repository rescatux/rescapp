Change Gnu/Linux password - Rescapp - Documentation
  • Step 1

    Rescapp searches for partitions that have on them the following file: /etc/issue based on information found in the /proc/partitions file.
  • Step 2

    If such partitions have been found, Rescapp then displays the partitions that have on them the following file: /etc/issue. At the same time you are prompted:

    Which partition is your main GNU/Linux?

    You are supposed to choose your main GNU/Linux distribution (you will probably only see one) and then press the OK button.

  • Step 3

    You are prompted:

    Which user?

    Select the desired user from the list.

  • Step 4

    You are prompted:

    Enter password (user)

    Use a temporary password please. You will change it later when you are logged into your system.

    • Step 5

    Rescapp tries to change the password for the selected user.
  • Step 6

    You are informed about the password change process.

    You either get:

    Password was changed OK! :). We recommend that you change it once you're back in your system to use your systems default encryption

    or

    Failure! Password was not changed. Something went wrong! :(

  • Step 7

    A log file called chpasswd.log.txt can be found in the Desktop logs folder.
