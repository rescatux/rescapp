File System Check (Forced Fix) - Rescapp - Documentation
  • Step 1

    Rescapp searchs for all the partitions based on information found in the /proc/partitions file. If the partition contains an: /etc/issue file it will also obtain the distribution name.
  • Step 2

    Rescapp displays the partitions found on Step 1.

    You are prompted:

    Which partition?

    You are supposed to choose whatever partition you want to force the filesystem check on and press the OK button.

    Explanation of description is:
    • Distribution name: Your distribution root partition has been detected and you are seeing its name.
    • Not detected: The partition can be mounted but there is not an /etc/issue file on it. Probably a non root partition.
    • Can't mount:: The partition cannot be mount. Maybe it is the partition you want to fix, another one that Rescatux cannot mount, or even a SWAP partition.

  • Step 3

    While the fsck process is being run the following message is displayed:Running forced filesystem check and fix

  • Step 4

    You are informed about the fsck process.

    You either get:

    Success! Filesystem check with automatic fix was OK! :)

    or

    Filesystem check with automatic fix failed! :(

  • Step 5

    You should click OK on the Running forced filesystem check and fix window.
  • Step 6

    A log file called fsck.log.txt can be found in the Desktop logs folder.
