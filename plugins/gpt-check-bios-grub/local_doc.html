Check if there is a bios_grub partition on a GPT disk - Rescapp - Documentation

Do you actually need a bios_grub partition? Probably not but it depends. A bios_grub partition is usually needed if you use grub-pc on a GPT disk. If your computer has both a GPT disk and EFI boot support, you may be better off trying to install the grub-efi package. If you need to use grub-pc package on a GPT disk after all, this option let's you find if your targeted hard drive has a bios_grub partition or not. If you think that you need to create a bios_grub partition please ask on your distributiona usual support places. An option to help you to create a bios_grub partition is not on the short term Rescatux TODO list.
  • Step 1

    Rescapp searches for hard drives on your computer.
  • Step 2

    If such hard drives have been found, Rescapp displays a list of the hard drives. At the same time you are prompted:

    Which hard drive?

    You are supposed to choose the hard drive which has a GPT partition for which you want its bios_grub partition to be checked.
  • Step 3

    Rescapp with all the gathered information, is going to check if there is a bios_grub partition on your GPT hard drive.Please wait. Currently with (Rescatux 0.40b1), this step is time consuming but no Please wait message has been implemented.
  • Step 4

    You are informed about the bios_grub checking process.

    You either get:

    bios_grub partition was found! :) or

    or

    No bios_grub partition was not found. :(
  • Step 5

    A log file called gpt-check-bios-grub_log.txt can be found in the Desktop logs folder.
